# 🎨 Part 1: 설계 기초

## 1. 설계 철학 및 원칙

### 1.1 핵심 설계 철학: "신뢰 + 친근함"

#### 기존 의료앱의 문제점 분석

```
❌ 신뢰만 강조하는 앱
   문제: 차갑고 딱딱함
   결과: 높은 이탈율
   예시: 전문 용어 남발, 복잡한 UI

❌ 친근함만 강조하는 앱
   문제: 의료 신뢰도 낮음
   결과: 실제 사용 안 함
   예시: 게임화 과다, 가벼운 느낌

✅ 우리의 접근 (신뢰 + 친근함 동시 달성)
   장점: 편하면서도 안심
   결과: 높은 지속 사용성
   전략: 의료 정확성 + 대화형 UI
```

#### 구체적 적용 예시

```
신뢰 요소:
✓ 식약처 인증 마크 표시
✓ 의료진 검증 표시
✓ 데이터 보안 배지
✓ 국민건강보험공단 연동
✓ 전문학회 가이드라인 준수

친근함 요소:
✓ 캐릭터 기반 대화
✓ 부드러운 톤의 메시지
✓ 이모지 사용
✓ 격려와 축하
✓ 가족 중심 UI
```

### 1.2 5가지 설계 기둥

```
1️⃣ 신뢰 (Trust)
   의료 정보와 제품 추천의 정확성
   
   적용 방법:
   • 모든 의료 정보에 출처 표시
   • 식약처 인증 제품만 추천
   • 의료진 프로필 상세 표시
   • 데이터 암호화 명시
   • HIPAA/GDPR 준수 표시
   
   측정 지표:
   • 의료진 신뢰도 평가 4.5/5.0 이상
   • 데이터 보안 만족도 95% 이상
   • 정보 정확성 피드백 긍정 90% 이상
   
2️⃣ 친근함 (Warmth)
   가족의 건강을 함께 돌보는 따뜻함
   
   적용 방법:
   • AI 캐릭터의 공감 대화
   • "당신", "우리" 같은 포용적 언어
   • 부드러운 색상과 라운드 코너
   • 성취 시 축하 메시지
   • 실패 시 격려 메시지
   
   측정 지표:
   • 감정 만족도 4.7/5.0 이상
   • 재방문 의도 80% 이상
   • 가족 초대율 40% 이상
   
3️⃣ 간결함 (Simplicity)
   복잡한 건강정보를 쉽게
   
   적용 방법:
   • 카드 기반 레이아웃
   • 한 화면에 한 가지 주제
   • 3클릭 내 목표 달성
   • Progressive Disclosure
   • 시각적 요약 우선
   
   측정 지표:
   • 태스크 완료율 95% 이상
   • 평균 학습 시간 3분 이하
   • 오류율 2% 이하
   
4️⃣ 지속성 (Retention)
   매일 사용하고 싶은 경험
   
   적용 방법:
   • 스트릭 시스템
   • 일일/주간 목표 설정
   • 루틴 완료 알림
   • 주간 리포트 제공
   • 마일스톤 축하
   
   측정 지표:
   • DAU/MAU 비율 40% 이상
   • 7일 리텐션 60% 이상
   • 30일 리텐션 40% 이상
   
5️⃣ 투명성 (Transparency)
   데이터와 추천의 근거 명확히
   
   적용 방법:
   • 제품 추천 이유 설명
   • 데이터 사용처 명시
   • 개인정보 처리 투명 공개
   • 면책 조항 항상 표시
   • 금기 사항 강조
   
   측정 지표:
   • 데이터 투명성 만족도 90% 이상
   • 개인정보 불안도 5% 이하
   • 제품 추천 신뢰도 85% 이상
```

### 1.3 3가지 핵심 원칙

```
원칙 1: Clarity Over Beauty
"아름답지만 이해하기 어려운 UI"보다
"단순하지만 명확한 UI"를 선택

나쁜 예시:
┌────────────────────────────────┐
│ [복잡한 그래디언트 배경]       │
│ [여러 겹 겹쳐진 카드]          │
│ [화려한 애니메이션]            │
│ "무엇을 해야 하는지 모르겠어요"│
└────────────────────────────────┘

좋은 예시:
┌────────────────────────────────┐
│ 오늘의 할 일                   │
│ ━━━━━━━━━━━━━━━━━━━━━━       │
│ ✅ 물 마시기                   │
│ ⚪ 비타민 복용                 │
│ ⚪ 스트레칭                    │
│ "명확하고 즉시 이해 가능!"     │
└────────────────────────────────┘

적용 지침:
• 한 화면 한 목적
• 명확한 CTA 버튼
• 간단한 레이블
• 불필요한 장식 제거
```

```
원칙 2: Empathetic Language
기술 용어 대신 공감하는 언어

나쁜 예시:
❌ "데이터 수집 중"
❌ "오류가 발생했습니다"
❌ "인증에 실패했습니다"
❌ "세션이 만료되었습니다"

좋은 예시:
✅ "당신의 건강 데이터를 기다리고 있어요"
✅ "앗, 문제가 생겼어요. 다시 시도해볼까요?"
✅ "다시 로그인이 필요해요"
✅ "잠시 자리를 비우셨네요. 안전을 위해 다시 로그인해주세요"

적용 지침:
• "당신", "우리" 사용
• 감정 표현 (앗, 와, 좋아요)
• 질문형 종결 (볼까요?, 해볼까요?)
• 긍정적 프레이밍
```

```
원칙 3: Progressive Disclosure
필요한 정보만 보이고, 깊이는 터치로

레벨 1 (첫 화면):
┌────────────────────────────────┐
│ 💓 심박수                      │
│ 72 BPM ✓ 정상                  │
│          [더보기 >]            │
└────────────────────────────────┘

레벨 2 (터치 후):
┌────────────────────────────────┐
│ 💓 심박수 상세                 │
│ ━━━━━━━━━━━━━━━━━━━━━━       │
│ 현재: 72 BPM                   │
│ 최근 7일 평균: 70 BPM          │
│ 정상 범위: 60-100 BPM          │
│ [차트 보기 >]                  │
└────────────────────────────────┘

레벨 3 (차트):
┌────────────────────────────────┐
│ [상세 차트]                    │
│ [분석 리포트]                  │
│ [AI 인사이트]                  │
└────────────────────────────────┘

적용 지침:
• 첫 화면은 핵심만
• 터치 시 더 많은 정보
• 3단계 이상 깊이 지양
• 뒤로가기 항상 가능
```

### 1.4 차별화 포인트

#### 1.4.1 vs 기존 의료앱

```
기존 의료앱:
• 단방향 정보 제공
• 텍스트 중심
• 개인 중심
• 높은 진입 장벽
• 유료 위주

음성 AI 건강주치의:
• 양방향 대화형 상담
• 음성 + 시각 통합
• 가족 중심
• 낮은 진입 장벽 (무료 기능 충실)
• 무료로 시작, 자연스러운 전환
```

#### 1.4.2 핵심 경쟁력

```
1. 캐릭터 기반 개인화
   → 단순 AI가 아닌 "의료진 친구"
   → 각기 다른 전문성과 성격
   → 사용자 선호도 반영
   
   예시:
   • 김소영 (가정의학과): 따뜻하고 꼼꼼한
   • 박민지 (정신건강의학과): 공감적이고 차분한
   • 이준호 (정형외과): 활동적이고 명확한

2. 무료 기본 기능의 완성도
   → 루틴 + 일기로 습관 형성
   → 70% 이상 무료 사용자 유입
   → 자연스러운 구독 전환 (5-8%)
   
   무료 기능:
   • 일일 5회 음성 상담
   • 건강 루틴 (무제한)
   • 감사 일기 (무제한)
   • 건강정보 검색 (무제한)
   • 웨어러블 연동 (기본)

3. 음성 우선 디자인
   → 핸즈프리 사용 가능
   → 실시간 음성 파형 피드백
   → 자동차/이동 중 사용 최적화
   
   기술:
   • OpenAI Realtime API
   • 실시간 STT/TTS
   • 음성 감정 분석
   • 자연스러운 대화 흐름

4. 신뢰 + 온기의 균형
   → 의료 정보 정확성
   → 인간적인 격려와 지지
   → 전문성과 접근성 양립

5. 공식 의료데이터 기반
   → 국민건강보험공단 연동
   → 전문학회 가이드라인
   → 의료진 검증 시스템
   → RAG 기반 지식 베이스

6. 다중 웨어러블 지원
   → Terra API로 150+ 기기 지원
   → Apple Watch, Fitbit, Garmin...
   → 플랫폼 제약 없음
   → 실시간 데이터 동기화
```
