## 🆕 주요 신규 기능

### 1. AI 기반 건강기능식품 추천

#### 핵심 특징
```
✅ 상담 중 자연스러운 추천
   - 증상/건강 목표에 맞는 제품
   - 사용자 동의 후 추천 표시
   - 강요하지 않는 선택적 제안

✅ RAG 기반 지능형 매칭
   - 증상 → 성분 → 제품 매칭
   - 임상 연구 기반 추천
   - 유사도 검색 알고리즘

✅ 개인화 안전성 검사
   - 나이, 성별, 건강 상태 고려
   - 만성질환 금기 성분 필터링
   - 약물 상호작용 확인
   - 개인별 주의사항 표시
```

#### 추천 프로세스
```
사용자: "요즘 눈이 피로해요"
   ↓
AI: "화면 노출이 많으시군요. 루테인 같은 성분이 도움될 수 있어요.
     💡 검증된 눈 건강 제품을 추천해드릴까요?"
   ↓
[네, 추천받을게요] 클릭
   ↓
제품 카드 3개 표시 (캐러셀)
   ├─ 제품 이미지
   ├─ 식약처 인증 마크
   ├─ 주요 성분 및 함량
   ├─ 💚 "엄마님께 적합" (개인화)
   ├─ 가격 및 할인율
   ├─ 별점 및 리뷰 수
   └─ [쿠팡/iHerb에서 보기] 버튼
   ↓
제품 상세 보기 (선택)
   ├─ 성분 상세 정보
   ├─ 임상 연구 결과
   ├─ 복용법 및 주의사항
   ├─ 개인화 적합성 분석
   └─ 사용자 리뷰
   ↓
구매 링크 클릭
   → Phase 1: 외부 쇼핑몰
   → Phase 2: 자체 쇼핑몰
```

### 2. 제품 데이터베이스 & RAG 시스템

#### 제품 데이터 구조
```
PostgreSQL (관계형 데이터)
├─ products (제품 기본 정보)
├─ product_ingredients (성분 정보)
├─ product_certifications (인증 정보)
├─ product_contraindications (금기 사항)
├─ product_drug_interactions (약물 상호작용)
├─ product_purchase_links (구매 링크)
└─ product_clicks/purchases (추적)

Pinecone/Weaviate (벡터 데이터)
├─ 제품 설명 임베딩
├─ 성분 임베딩
├─ 증상 임베딩
└─ 임상 연구 임베딩
```

#### 제품 검증 프로세스
```
1️⃣ 제품 등록 (관리자)
   └─ 9개 필드 입력 (기본정보~추천 키워드)

2️⃣ 자동 검증
   ├─ 필수 정보 확인
   ├─ 성분 데이터 형식 확인
   ├─ 이미지 품질 확인
   └─ 중복 제품 확인

3️⃣ 의료진 검수 (영양사/약사)
   ├─ 성분 정보 정확성
   ├─ 기능성 주장 적절성
   ├─ 안전성 정보 충분성
   └─ 금기사항 명확성

4️⃣ 승인/반려
   ├─ 승인 → 활성화 + RAG 업데이트
   └─ 반려 → 수정 요청 (상세 사유)

5️⃣ RAG 시스템 업데이트
   ├─ 제품 임베딩 생성
   ├─ 벡터 DB 저장
   └─ 검색 인덱스 갱신
```

### 3. 관리자 제품 관리 시스템

#### 대시보드
```
┌─────────────────────────────────┐
│  제품 관리 대시보드              │
├─────────────────────────────────┤
│  📊 통계                         │
│  • 총 제품: 1,234개             │
│  • 활성 제품: 1,180개           │
│  • 검수 대기: 54개              │
│  • 오늘 클릭: 2,345회           │
│  • 오늘 구매: 87건              │
│  • 전환율: 3.7%                 │
│                                 │
│  [+ 새 제품 추가]               │
│                                 │
│  제품 목록                       │
│  [검색/필터/정렬]               │
└─────────────────────────────────┘
```

#### 제품 등록 단계 (7단계)
```
1️⃣ 기본 정보
   - 제품명, 브랜드, 카테고리
   - 제품 설명

2️⃣ 성분 정보
   - 성분명, 함량, 1일 권장량
   - 원료, 기능

3️⃣ 인증 정보
   - 식약처 기능성 인증
   - GMP, 임상시험 등
   - 인증서 파일 업로드

4️⃣ 안전성 정보
   - 복용법
   - 최소 연령, 임산부/수유부 안전성
   - 금기 사항 (만성질환)
   - 약물 상호작용

5️⃣ 가격 및 판매
   - 정가, 할인가
   - 구매 링크 (쿠팡, iHerb 등)
   - 제휴 코드

6️⃣ 이미지 및 비디오
   - 제품 이미지 (최대 5개)
   - 소개 영상 (YouTube)

7️⃣ 추천 알고리즘
   - 추천 키워드 (증상/목표)
   - 추천 우선순위
```

## 💡 FAQ

### Q1: 왜 제품 추천 기능이 필요한가요?
```
A: AI 상담은 조언을 주지만, 실천 방법은 부족합니다.
   
   사용자는 "그래서 뭘 먹어야 하나요?"라는 질문을 자주 합니다.
   검증된 제품을 추천함으로써:
   
   1. 사용자에게 실질적인 가치 제공
   2. 믿을 수 있는 제품 선택 지원
   3. 앱 수익 다각화 (커미션)
   4. 사용자 참여도 증가
```

### Q2: 추천 제품은 안전한가요?
```
A: 네, 철저한 검증을 거칩니다:
   
   1. 식약처 기능성 인증 제품만
   2. 의료진(영양사, 약사) 검수
   3. 성분 정보 투명 공개
   4. 개인 건강 상태 고려
   5. 금기 사항 자동 필터링
   
   + 면책 조항 항상 표시
```

### Q3: Phase 1과 Phase 2의 차이는?
```
A: Phase 1 (출시): 제휴 링크
   - 쿠팡, iHerb 등 외부 쇼핑몰
   - 클릭 수수료 + 구매 커미션
   - 빠른 구현 (4주)
   
   Phase 2 (6개월 후): 자체 쇼핑몰
   - 장바구니, 결제, 정기 배송
   - 더 높은 마진 (15-30%)
   - 포인트, 쿠폰 시스템
```

### Q4: 제품 추천이 강제적이지 않나요?
```
A: 아니요, 자연스럽고 선택적입니다:
   
   1. 상담 맥락에 맞게 추천
   2. 사용자 동의 후 표시
   3. "괜찮아요" 선택 가능
   4. 강요하지 않는 톤
   5. 면책 조항 명시
```

### Q5: RAG 시스템은 어떻게 작동하나요?
```
A: 벡터 유사도 검색:
   
   사용자 질문 → 임베딩 생성 → 유사 증상/성분 검색
      → 제품 매칭 → 개인화 필터링 → Top 3 추천
   
   예: "눈이 피로해요"
      → [루테인, 지아잔틴] 성분 매칭
      → 해당 성분 포함 제품 검색
      → 나이/질환 고려 필터링
      → 3개 제품 추천
```
